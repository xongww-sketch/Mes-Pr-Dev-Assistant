# 需求分析 SOP

> 版本：1.0 | 最近更新：2026-04-14
> 用途：MES需求分析、评估、PRD撰写的标准流程

---

## 1. 需求接收

### 1.1 信息来源
- 飞书群聊（MES·EL、Mes开发需求对接）
- 私聊
- 会议
- 工单系统

### 1.2 信息收集
收到需求时，确认以下信息：

```
【必填】
- 需求名称：一句话描述
- 提出人：谁提出的
- 业务背景：为什么需要这个需求
- 期望结果：做成什么样算完成

【选填】
- 优先级期望：提出人认为的优先级
- 期望上线时间：有没有时间要求
- 相关文档：有没有参考文档
```

## 2. 需求评估

### 2.1 边界判断

**第一步：判断是否在MES边界内**

参考 `_core/mes-domain-knowledge.md` 中的系统边界：

| 在MES边界内 ✅ | 不在MES边界 ❌ |
|---------------|---------------|
| 工单管理 | 生产计划制定（APS/ERP） |
| 工序/过站 | 库存账务管理（WMS） |
| SN管理 | 订单管理（OMS） |
| 质量数据采集 | 财务核算（SAP） |
| 物料绑定 | 采购管理（SRM） |
| 设备状态监控 | 人力资源管理 |
| 报工/产量统计 |  |
| 全流程追溯 |  |

**判断结果**：
- 在边界内 → 继续评估
- 不在边界内 → 退回或转交对应系统负责人

### 2.2 必要性判断

**按 `_core/decision-framework.md` 三条原则评估**：

```
原则1：这个对系统有没有必要？
- 是否解决实际问题？
- 是否影响核心业务流程？
- 有没有替代方案？

原则2：以后好不好维护？
- 实现方案是否清晰？
- 新人能否快速理解？
- 修改影响范围是否可控？

原则3：设计符不符合长线发展？
- 是否能支撑未来1-2年业务增长？
- 是否符合系统架构演进方向？
- 是不是临时方案？
```

### 2.3 影响范围评估

```
- 涉及哪些模块？（工单/SN/工序/质量/...）
- 涉及哪些n8n workflow？（查 _core/n8n-process-map.md）
- 涉及哪些外部系统？（SAP/OMS/WMS/...）
- 影响哪些用户？（产线/仓库/质检/...）
```

### 2.4 工作量评估

```
- 前端：有/无
- 后端：有/无
- n8n：新增/修改workflow
- 数据库：新增/修改表结构
- 测试：需要/不需要
- 文档：需要/不需要

预估工作量：S(1天) / M(3天) / L(1周) / XL(2周+)
```

### 2.5 优先级判定

| 优先级 | 条件 | 响应时间 |
|--------|------|---------|
| P0 | 阻断性Bug、产线停线 | 立即 |
| P1 | 核心功能缺失、影响业务 | 本周 |
| P2 | 重要但不紧急 | 本月 |
| P3 | 优化建议、体验提升 | 排期 |

## 3. 需求决策

### 3.1 决策矩阵

```
                    必要性高
                      │
              好维护  │  不好维护
            ┌─────────┼─────────┐
            │  ✅ 接  │ ⚠️ 重设计│
  长线发展  │  P1/P2  │  重新设计 │
            ├─────────┼─────────┤
            │  ⚠️ 接  │  ❌ 退回 │
  非长线发展│  临时方案│  没必要  │
            └─────────┼─────────┘
                      │
                    必要性低
```

### 3.2 决策输出

```markdown
## 需求评估结论

**需求名称**：
**评估结果**：✅ 接受 / ⚠️ 需要重新设计 / ❌ 退回

**理由**：
- 边界判断：在/不在MES边界内
- 必要性：___
- 可维护性：___
- 长线发展：___

**优先级**：P0/P1/P2/P3
**预估工作量**：S/M/L/XL
**负责人**：
**预计完成时间**：
```

## 4. PRD 撰写

### 4.1 PRD 模板

```markdown
# [需求名称] PRD

## 1. 产品概览
### 1.1 业务范围
### 1.2 功能概述
### 1.3 名词解释

## 2. 流程详述
### 2.1 流程图
### 2.2 流程描述

## 3. 产品设计
### 3.1 系统架构
### 3.2 功能详述
#### 3.2.1 [功能模块1]
##### 3.2.1.1 单据设计
##### 3.2.1.2 关键控制
##### 3.2.1.3 交互设计

## 4. 数据设计
### 4.1 数据库表
### 4.2 接口设计

## 5. 异常处理
### 5.1 异常场景
### 5.2 处理方案

## 6. 待确认事项
| # | 事项 | 负责人 | 状态 |
|---|------|--------|------|
| 1 |      |        |      |

## 7. 上线计划
### 7.1 上线步骤
### 7.2 回滚方案
```

### 4.2 PRD 撰写要点

- **流程图**：用表格描述流程步骤（部门/角色/流程描述/系统/数据/备注）
- **单据设计**：明确字段名称、类型、录入方式、说明
- **关键控制**：明确控制项和控制规则
- **交互设计**：描述操作场景和交互流程
- **待确认事项**：列出所有未决事项，明确负责人
