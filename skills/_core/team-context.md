## 3. 问题升级路径

### 3.1 Bug处理升级

```
Level 1: Agent自助处理
  → 查知识库 → 查n8n流程地图 → 给出诊断建议

Level 2: 需要人工确认
  → 通知熊旺(Jackson) → 确认处理方案

Level 3: 需要开发介入
  → 通知对应开发工程师（王爵/黄启立）

Level 4: 需要架构/运维介入
  → 通知王晓虎(Hunter Wang)

Level 5: 需要跨部门协调
  → 通知熊旺 + 相关部门负责人
```

### 3.2 紧急程度判断

| 程度 | 定义 | 响应时间 | 示例 |
|------|------|---------|------|
| P0 紧急 | 产线停线、全部workflow不可用 | 立即 | n8n worker全挂、数据库不可用 |
| P1 高 | 核心功能不可用、影响部分产线 | 30分钟内 | SN生成失败、绑定失败 |
| P2 中 | 非核心功能异常、有替代方案 | 2小时内 | 报表查询慢、邮件未发送 |
| P3 低 | 体验问题、优化建议 | 当天 | UI显示问题、文档更新 |

## 5. 沟通规范

### 5.1 群聊规则（从群聊蒸馏）
- 复杂问题建「日期+主题」子群
- 工作沟通群内进行，紧急私聊需补群
- 新需求必附飞书文档再讨论
- 数据佐证僵持观点，2小时未决即升级

### 5.2 问题反馈格式
```
【问题类型】Bug/需求/咨询
【影响范围】产线/工单/SN/全部
【紧急程度】P0/P1/P2/P3
【问题描述】...
【复现步骤】...
【期望结果】...
【实际结果】...
```
